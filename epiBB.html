Machine learning: digit recognition

Machine learning (aka statistical learning): using data to fit a statistical model to make predictions.

with machine learning

  • build a training set
  • use it to build a statistical model
  • use the model to make predictions on new data

This example uses the bagging algorithm from the ipred package in R, with a training set of 10,000 digits:

library(ipred)
bagmodel    <- bagging(label~., data=trainingdata)
predictions <- predict(bagmodel, newdata=testdata)

How this works

The algorithm fits a statistical model to the data provided in the individual pixels.

explanation versus prediction

There are 2 basic types of statistical models: explanatory models and predictive models

explanatory models

  • purpose is to estimate 1 or a few coefficients
  • parsimony is good
  • confounders, matching variables, etc, are considered 'nuisance variables'
  • the paramount concern is the interpretability of the main coefficients

predictive models

  • purpose is only to make good predictions
  • they may be arbitrarily complex
  • individual coefficients are generally not meaningful on their own

examples of machine learning algorithms

  • bagging (bootstrap aggregating): many bootstrap samples of the original data are taken, a decision tree is fit to each, and the trees are averaged
  • boosting: decision trees are grown sequentially; after the first tree, each tree is built using the residuals from the previous tree; then the trees are averaged
  • random forests: similar to bagging, except that each decision tree is built from a random subset of the predictors.
  • svm (support vector machines): the details are somewhat complicated
  • neural networks: the details are somewhat complicated

two types of machine learning

supervised learning

  • uses a training set
  • builds a prediction model

unsupervised learning

  • no training set
  • finds clusters or patterns in the data

rules of thumb

3 features determine how well a predictive model works

  • size of the training set (bigger is better)
  • the predictors used in the model (i.e. the characteristics of the training data)
  • the algorithm used to build the model

Record Linkage

the basic unit of analysis is a record pair:

dob first name MI last name ssnL4 sex first.sdx last.sdx subname

03/09/1944 JOSE SANCHEZORTIZ M J200 J522 SANCHEZORTIZ
agreement: 1 . 0 . 1 1 1 1
08/15/1975 MELISSA R PETERS 4393 F M420 P362 PETERS
08/15/1975 MELISSA M PETERSON 3902 F M420 P362 PETERSON
agreement: 1 0 0 0 1 1 1 1

A large number of record pairs need to be compared:

  • \(n_1 \times n_2\) for record linkage,
  • \(n(n-1)/2\) for deduplication

For example, the 2009-2013 CHARS files have 3,713,485 records, so there are 13,789,967,131,740 record pairs.

Blocking

To reduce the number of record pairs, we require agreement on one or more fields. This is called blocking.

Good blocking variables have these features:

  • accurate and complete recording
  • consistency
  • many categories

Methods for record linkage

Deterministic: rule-based; requires specified fields to agree

Probabilistic: uses a similarity measure between records

Machine learning: uses a statistical model to predict links

Most record linking projects use a combination of deterministic (through blocking) and probabilistic methods, but machine learning may work better.

Probabilistic linking

Fellegi and Sunter developed a mathematical framework for probabilistic linking in 1969.

For a record pair, each field gets a positive weight if its values agree, and a negative weight if its values don't agree. The weights are summed to produce the weight for the record pair.

linkage by machine learning

  • Generate a training set of record pairs. Some software can create a training set that provides examples of every possible permutation of field matches and mismatches.
  • Score it manually.
  • Use a machine learning algorithm to create a statistical model to predict links.
  • Apply the model to the unlinked record pairs.

linkage software

RecordLinkage package in R

pros:

  • very flexible; user can add custom functions
  • provides a full suite of methods:
    • several different ways to cmopute probabilistic weights
    • supervised learning
    • unsupervised learning

cons:

  • requires knowledge of R
  • limited by available RAM; cannot do really large files
  • does not provide a function for combining linked pairs
